# APS改造关键经验 - 标准产能设置

## 🚨 最重要的7个问题

### 1. 使用正确的数据库表 ⭐⭐⭐⭐⭐
```bash
# 检查数据库表
mysql -h localhost -u root -p aps_db -e "SHOW TABLES LIKE '%line%';"

# 查看表结构
mysql -h localhost -u root -p aps_db -e "DESC production_lines;"

# 对比表结构
# ❌ sys_work_center（工作中心表）
# ✅ production_lines（产线表）
```

### 2. 添加缺失的数据库字段 ⭐⭐⭐⭐⭐
```sql
-- 添加字段
ALTER TABLE production_lines
ADD COLUMN standard_capacity_area DECIMAL(10,2)
COMMENT '标准产能面积(㎡)'
AFTER standard_capacity;

-- 验证字段
DESC production_lines;
```

### 3. 更新Model文件 ⭐⭐⭐⭐⭐
```javascript
// 文件：aps-backend/src/models/ProductionLine.js
standard_capacity_area: {
  type: DataTypes.DECIMAL(10, 2),
  allowNull: true,
  comment: '标准产能面积(㎡)'
}
```

### 4. 字段名映射 ⭐⭐⭐⭐
```javascript
// 前端 → 后端映射
{
  standardCapacity → standard_capacity,
  standardCapacityArea → standard_capacity_area,
  mainEquipments → main_equipment (数组↔字符串)
}

// 返回数据时映射
const productionLine = {
  lineId: line.line_id,
  lineName: line.line_name,
  standardCapacity: line.standard_capacity,
  standardCapacityArea: line.standard_capacity_area,
  mainEquipments: line.main_equipment ? line.main_equipment.split(',') : []
}
```

### 5. V2 API适配 ⭐⭐⭐⭐
```javascript
// API路径
❌ /api/production-lines
✅ /api/v2/production-lines

// 响应格式
❌ { code: 'SUCCESS', data: {...} }
✅ { success: true, data: {...} }

// 兼容处理
if (result.code === 'SUCCESS' || result.success === true) {

```

## 🔍 常见错误及解决方案

### 错误1：400 Bad Request
**原因**：字段名不匹配或缺少必需字段
**解决**：
1. 检查前端发送的字段名
2. 检查后端期望的字段名
3. 检查数据库表结构
4. 确保字段映射正确

### 错误2：字段保存失败
**原因**：数据库表缺少该字段
**解决**：
1. 添加缺失的数据库字段
2. 更新Model文件
3. 重启后端服务

### 错误3：列表显示不完整
**原因**：字段映射错误或字段未返回
**解决**：
1. 检查后端返回的数据结构
2. 确保所有字段都已映射
3. 检查数据库字段是否有值

### 错误4：模式切换不生效
**原因**：Store文件导入错误
**解决**：
1. 检查所有Store文件
2. 确保从 `api-unified` 导入
3. 刷新前端页面

### 错误5：主键冲突
**原因**：新增时没有生成主键
**解决**：
1. 在新增表单中自动生成主键
2. 使用时间戳确保唯一性
3. 检查主键格式符合约束

## 💡 最佳实践

### 数据库字段管理
1. **命名规范**：使用下划线命名（snake_case）
2. **字段类型**：根据业务需求选择合适的类型
3. **注释完整**：每个字段都要有清晰的注释
4. **及时更新**：字段变更后立即更新Model文件

### API设计
1. **版本管理**：明确使用V1还是V2
2. **路径规范**：统一使用 `/api/v2/` 前缀
3. **响应格式**：统一使用 `{ success: true }` 格式
4. **兼容处理**：支持V1和V2两种格式

### 前端开发
1. **导入规范**：统一从 `api-unified` 导入
2. **字段映射**：前端使用驼峰命名
3. **数据转换**：数组与字符串相互转换
4. **主键生成**：新增时自动生成唯一ID

### 测试验证
1. **全面测试**：测试所有CRUD操作
2. **数据验证**：验证数据库数据正确性
3. **模式切换**：测试两种模式都能正常工作
4. **边界测试**：测试异常情况和边界条件
