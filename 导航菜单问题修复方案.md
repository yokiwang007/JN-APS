# 导航菜单问题修复方案

## 📊 问题总结

### 核心问题
系统使用了**两种不同的数据加载机制**,导致部分菜单点击后没有数据或显示空白。

### 机制对比

#### 机制1: 新机制 (22个页面)
- **数据源**: `initializeExtendedData()`
- **特点**:
  - 数据完整,包含11个数据表
  - 数据关联性强,上下游关系清晰
  - 使用下划线命名 (如 `erp_sales_orders`)
  - 数据量: 4,485条记录

#### 机制2: 旧机制 (13个页面)
- **数据源**: Pinia Store + API (`getOrders()` 等)
- **特点**:
  - 数据来源: `initMockData()` (旧的mock数据)
  - 数据较简单,只有orders和panels
  - 使用驼峰命名 (如 `orders`)
  - 数据量: 较少

## ✅ 修复方案

### 方案A: 统一使用新机制 (推荐)

**优点**:
- 数据完整,关联关系清晰
- 便于维护和扩展
- 统一的数据标准

**缺点**:
- 需要修改22个页面
- 工作量较大

**实施步骤**:

#### 步骤1: 修改API文件
将 `api.js` 中的数据源改为使用 `initializeExtendedData()`

```javascript
// 修改前
import { initMockData } from './mock'

// 修改后
import { initializeExtendedData } from './mock-extended'
```

#### 步骤2: 修改API函数
修改所有API函数,使用新的数据结构

```javascript
// 修改前
export const getOrders = async (params = {}) => {
  const data = getData()
  let orders = [...data.orders]
  // ...
}

// 修改后
export const getOrders = async (params = {}) => {
  const data = initializeExtendedData()
  let orders = [...data.erp_sales_orders]
  // ...
}
```

#### 步骤3: 修改字段映射
将驼峰命名改为下划线命名

```javascript
// 修改前
orderNo → orderNo
customerName → customerName

// 修改后
orderNo → order_id
customerName → customer_name
```

#### 步骤4: 测试所有页面
确保所有25个菜单都能正常工作

### 方案B: 更新Store数据源 (快速修复)

**优点**:
- 只需修改API文件
- 工作量小
- 不影响现有页面代码

**缺点**:
- 仍然使用旧的数据结构
- 数据关联性不如新机制

**实施步骤**:

#### 步骤1: 修改API文件
在 `api.js` 中添加使用新数据的函数

```javascript
import { initializeExtendedData } from './mock-extended'

// 添加新函数
export const getExtendedOrders = async (params = {}) => {
  await delay()
  const data = initializeExtendedData()
  let orders = [...data.erp_sales_orders]
  
  // 筛选、排序、分页逻辑
  // ...
  
  return success({ total, list, page, pageSize })
}
```

#### 步骤2: 修改Store
在Store中使用新的API函数

```javascript
// 修改前
const result = await getOrders({ ...filters.value, ...pagination.value })

// 修改后
const result = await getExtendedOrders({ ...filters.value, ...pagination.value })
```

#### 步骤3: 修改页面字段
修改页面模板,使用新的字段名

```vue
<!-- 修改前 -->
<el-table-column prop="orderNo" label="订单号" />
<el-table-column prop="customerName" label="客户名称" />

<!-- 修改后 -->
<el-table-column prop="order_id" label="订单号" />
<el-table-column prop="customer_name" label="客户名称" />
```

最终将所有页面都迁移到新机制
