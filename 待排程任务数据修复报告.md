# 待排程任务数据修复报告

## 📊 问题发现时间
2026-04-04

## ❌ 问题描述

### 症状
- 点击"待排程任务"菜单后,页面显示空白
- 没有显示任何待排程子订单数据
- 产能看板显示静态数据,未与实际数据关联

### 影响范围
- 待排程任务页面 (`/coarse-schedule/tasks`)
- 排程任务管理页面 (`/coarse-schedule/list`)
- 排程甘特图页面 (`/coarse-schedule/gantt`)

---

## 🔍 问题原因分析

### 根本原因

#### 问题1: 字段名不一致
在 `CoarseTasks.vue` 中,代码使用 `data.apsProductionOrders` 访问数据,但实际返回的对象中字段名应为 `data.aps_production_orders`(下划线命名)。

```javascript
// 错误代码 (第243行)
pendingTasks.value = data.apsProductionOrders
  .filter(po => po.status === '待排程')
```

#### 问题2: 数据状态分布不合理
在 `mock-extended.js` 中,所有生产子订单的状态都被硬编码为"待排程",导致:
- 没有已排程的数据
- 没有已合并的数据
- 没有生产中的数据
- 无法模拟真实的业务流程

```javascript
// 错误代码 (第286行)
status: '待排程',  // 所有订单都是待排程状态
```

#### 问题3: 排程任务生成逻辑错误
在 `generateApsScheduleTasks` 函数中,只为"待排程"状态的订单生成排程任务,这违反了业务逻辑:
- 排程任务应该是已排程后的数据
- 待排程的订单不应该有排程任务

```javascript
// 错误代码 (第335行)
if (po.status === '待排程') {  // 应该是 '已排程'
```

#### 问题4: 产能数据未关联
在 `CoarseTasks.vue` 中,产能数据使用硬编码的静态数据,未与实际的工作中心和排程任务关联。

```javascript
// 错误代码 (第248-285行)
capacityData.value = [
  {
    line_id: 'LINE001',
    line_name: '柜体A线',
    // 硬编码的静态数据
  }
]
```

## ✅ 解决方案

### 修复1: 修正字段名
将 `data.apsProductionOrders` 改为 `data.aps_production_orders`

### 修复2: 优化数据状态分布
在生产子订单生成时,随机分配不同状态:
- 待排程: 20%
- 已排程: 30%
- 已合并: 20%
- 生产中: 20%
- 已完成: 10%

```javascript
const statuses = ['待排程', '已排程', '已合并', '生产中', '已完成']
const status = statuses[Math.floor(Math.random() * statuses.length)]
```

### 修复3: 修正排程任务生成逻辑
只为"已排程"状态的订单生成排程任务

```javascript
if (po.status === '已排程') {  // 修正: 待排程 → 已排程
  // 生成排程任务
}
```

### 修复4: 关联上游数据
在待排程任务列表中:
- 关联销售订单信息(客户名称、交期)
- 关联BOM零件信息(零件数量)

```javascript
pendingTasks.value = data.aps_production_orders
  .filter(po => po.status === '待排程')
  .map(po => {
    // 关联销售订单
    const order = data.erp_sales_orders.find(o => o.order_id === po.order_id)
    
    // 关联BOM零件
    const parts = data.cad_bom_parts.filter(p => p.order_id === po.order_id)
    
    return {
      ...po,
      customer_name: order?.customer_name || '',
      due_date: order?.due_date || '',
      total_part_count: parts.length
    }
  })
```

### 修复5: 关联产能数据
产能数据与工作中心和排程任务关联:
- 获取该产线已排程的任务数量
- 计算实际产能负荷

```javascript
capacityData.value = data.sys_work_centers.map(line => {
  const scheduledTasks = data.aps_schedule_tasks.filter(t => t.line_id === line.line_id)
  const scheduledCount = scheduledTasks.length * 10
  const remaining = line.daily_capacity - scheduledCount
  const loadRate = Math.min(100, Math.round((scheduledCount / line.daily_capacity) * 100))

  return {
    line_id: line.line_id,
    line_name: line.line_name,
    daily_capacity: line.daily_capacity,
    scheduled: scheduledCount,
    remaining: Math.max(0, remaining),
    load_rate: loadRate
  }
})
```

### 修复6: 增加显示列
在待排程任务列表中增加:
- 客户名称列
- 交期列
