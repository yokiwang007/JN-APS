## 🎯 7阶段业务流程

V2版本支持完整的7阶段业务流程：

1. **订单接收** - 接收销售订单（`/api/v2/erp-orders`）
2. **订单拆解** - 拆解为生产子订单（`/api/v2/orders/:orderId/decompose`）
3. **粗排程** - 产能粗排（`/api/v2/scheduling/coarse`）
4. **揉单合并** - 批次合并优化（`/api/v2/merging/merge`）
5. **细排程** - 详细排版计划（`/api/v2/scheduling/fine`）
6. **生产执行** - 下发生产指令（`/api/v2/production/issue`）
7. **齐套管理** - 齐套与发货（`/api/v2/fulfillment/slots`）

## 📝 常见问题

### Q: 为什么使用V2版本？
A: V2版本是重新设计的后端，支持新的表结构和完整的7阶段业务流程，是目前推荐的版本。

### Q: 前端如何适配V2？
A: 前端已更新为调用V2 API路径，并兼容V2的响应格式。无需额外配置。

### Q: 启动报错端口被占用？
A: 检查是否有其他服务占用3002端口：
```bash
lsof -ti:3002
```

### Q: 数据库连接失败？
A: 确保 MySQL 服务运行，且环境变量配置正确：
```bash
# 检查MySQL服务
mysql -h localhost -u root -pjienor0803 -e "SELECT 1"

# 检查数据库
mysql -h localhost -u root -pjienor0803 -e "USE aps_db; SHOW TABLES;"
```

### Q: V1版本还在吗？
A: V1版本已废弃，不再使用。所有功能已迁移到V2版本。
