## 🎯 核心功能实现

### 1. "先排程后揉单"核心逻辑 ✅

**实现方式**:
1. 粗排程阶段: 为生产子订单分配计划日期和产线
2. 揉单阶段: 根据时间窗规则将相近日期的订单合并
3. 细排程阶段: 对合并后的批次进行开料优化

**相关页面**:
- `/coarse-schedule/tasks` - 待排程任务
- `/batch/plan` - 批次规划
- `/schedule/optimization` - 开料优化

### 2. 倒排计划 ✅

**实现方式**:
- 基于订单交期倒推计划开工日期
- 考虑齐套缓冲期(可配置)
- 自动计算最迟开工日

**相关页面**:
- `/coarse-schedule/tasks` - 排程模式选择
- `/coarse-schedule/list` - 排程任务管理

### 3. 交期红线策略 ✅

**实现方式**:
- 监控订单交期和计划日期
- 达到最迟开工日时触发预警
- 支持强制下发和揉单下发

**相关页面**:
- `/batch/merge-rules` - 交期红线策略配置
- `/batch/plan` - 交期红线任务处理

### 4. 智能填缝 ✅

**实现方式**:
- 监控批次利用率
- 利用率低于阈值时自动触发
- 从标准件需求池、售后补件池等来源填充

**相关页面**:
- `/batch/merge-rules` - 智能填缝策略配置

### 5. 齐套管理 ✅

**实现方式**:
- 实时追踪订单齐套进度
- 货位分配和管理
- 超期预警(占用超过7天)
- 紧急订单优先处理

**相关页面**:
- `/fulfillment/dashboard` - 齐套看板
- `/fulfillment/slots` - 齐套货位管理

## 📊 数据流转关系

### 完整业务流程

```
1. 销售订单接收 (erp_sales_order)
   ↓
2. 订单拆解 (aps_production_order)
   ↓
3. 粗排程 (aps_schedule_task)
   ↓
4. 揉单批次 (aps_merge_batch + aps_batch_detail)
   ↓
5. 细排程 (cam_cutting_pattern)
   ↓
6. 生产执行 (生产工单 + mes_wip_tracking)
   ↓
7. 齐套管理 (mes_sorting_slot)
```

### 数据关联示例

- 订单拆解 → 粗排程: `aps_production_order.order_id` → `erp_sales_order.order_id`
- 粗排程 → 揉单: `aps_batch_detail.task_id` → `aps_schedule_task.task_id`
- 揉单 → 细排程: `cam_cutting_pattern.batch_id` → `aps_merge_batch.batch_id`
- 细排程 → 生产执行: `mes_wip_tracking.work_order_id` → 生产工单
- 生产执行 → 齐套: `mes_sorting_slots.order_id` → `erp_sales_order.order_id`
