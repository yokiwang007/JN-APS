# 数据字段命名修复说明

## 📊 问题说明

### 问题描述
Vite的热更新功能会自动检测文件变化并更新页面,但有时需要手动刷新浏览器才能看到最新效果。

### 修复内容
统一了 `mock-extended.js` 中 `initializeExtendedData()` 函数返回的数据对象字段命名:
- 从驼峰命名改为下划线命名
- 与数据库表名保持一致

### 修改的文件

#### 1. `/Users/yoki/IDEProjects/JN-APS/aps-frontend/src/utils/mock-extended.js`

**修改前**:
```javascript
return {
  erpSalesOrders,
  cadBomParts,
  apsProductionOrders,
  sysWorkCenters,
  apsScheduleTasks,
  apsMergeBatches,
  apsBatchDetails,
  camCuttingPatterns,
  mesProductionOrders,
  mesWipTracking,
  mesSortingSlots,
  mesFulfillmentDetails
}
```

**修改后**:
```javascript
return {
  erp_sales_orders: erpSalesOrders,
  cad_bom_parts: cadBomParts,
  aps_production_orders: apsProductionOrders,
  sys_work_centers: sysWorkCenters,
  aps_schedule_tasks: apsScheduleTasks,
  aps_merge_batches: apsMergeBatches,
  aps_batch_details: apsBatchDetails,
  cam_cutting_patterns: camCuttingPatterns,
  mes_production_orders: mesProductionOrders,
  mes_wip_tracking: mesWipTracking,
  mes_sorting_slots: mesSortingSlots,
  mes_fulfillment_details: mesFulfillmentDetails
}
```

#### 2. `/Users/yoki/IDEProjects/JN-APS/aps-frontend/src/views/dashboard/Dashboard.vue`

**修改的字段**:
- `data.erpSalesOrders` → `data.erp_sales_orders`
- `data.mesSortingSlots` → `data.mes_sorting_slots`

### 修改原因
1. **统一命名规范**: 与数据库表名保持一致,使用下划线命名
2. **避免混淆**: 驼峰命名和下划线命名混用容易出错
3. **提高可维护性**: 统一的命名规范便于后续维护

1. 打开浏览器开发者工具 (F12)
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"
