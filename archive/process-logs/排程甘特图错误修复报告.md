# 排程甘特图错误修复报告

## 📊 问题发现时间
2026-04-04

## ❌ 问题描述

### 错误信息
```
Uncaught TypeError: Cannot read properties of undefined (reading 'map')
    at updateGanttChart (CoarseGantt.vue:189:24)
    at handleRefresh (CoarseGantt.vue:240:3)
```

### 问题说明
在更新甘特图时,尝试对 `undefined` 调用 `map` 方法,导致错误。

---

## 🔍 问题原因分析

### 根本原因

#### 1. 数据未初始化
`productionLines.value` 在某些情况下可能是 `undefined` 或空数组,导致后续的 `filter` 和 `map` 操作失败。

#### 2. 缺少安全检查
代码没有对数据的有效性进行检查,直接进行操作。

#### 3. 异步调用时机问题
虽然 `initializeData()` 是同步的,但在某些情况下,`updateGanttChart()` 可能在数据完全加载之前被调用。

---

## ✅ 解决方案

### 修复方法

#### 1. 添加安全检查
在 `updateGanttChart` 函数中添加对 `lines` 数组的安全检查:

```javascript
// 按产线分组
const lines = selectedLine.value
  ? productionLines.value.filter(l => l.line_id === selectedLine.value)
  : productionLines.value

// 安全检查:如果产线数据为空,不渲染图表
if (!lines || lines.length === 0) {
  console.warn('⚠️ 产线数据为空,无法渲染甘特图')
  return
}
```

#### 2. 添加默认值
为 `ganttTasks.value` 添加默认值,防止 `undefined`:

```javascript
// 筛选任务
let filteredTasks = ganttTasks.value || []
```

#### 3. 添加调试信息
添加控制台日志,便于排查问题:

```javascript
console.log('📊 更新甘特图')
console.log('  - 产线数据:', productionLines.value?.length || 0, '条')
console.log('  - 任务数据:', ganttTasks.value?.length || 0, '条')
console.log('  - 选中的产线:', selectedLine.value)
```
