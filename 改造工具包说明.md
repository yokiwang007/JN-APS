# APS改造工具包使用说明

## 📦 工具包内容

我们已经为后续业务模块改造准备了完整的工具包，包含以下文件：

### 1. **APS改造最佳实践.md** 📖
- **内容**：完整的改造指南，包含详细的步骤、代码示例、常见问题解决方案
- **用途**：改造过程中的参考文档
- **特点**：
  - 详细的改造步骤说明
  - 完整的代码示例模板
  - 常见问题和解决方案
  - 改造检查清单
  - 快速开始模板

### 2. **快速参考卡片.md** ⚡
- **内容**：5分钟快速开始指南
- **用途**：快速查阅关键代码和步骤
- **特点**：
  - 精简的步骤说明
  - 关键代码片段
  - 常见错误快速修复
  - 一句话总结

### 3. **改造检查清单.sh** ✅
- **内容**：交互式改造检查清单
- **用途**：改造过程中逐项检查
- **特点**：
  - 分类检查项（后端、前端、测试）
  - 可自定义模块名称
  - 提供测试命令参考

### 4. **check-services.sh** 🔧
- **内容**：服务状态检查脚本
- **用途**：检查前后端服务和数据库连接状态
- **特点**：
  - 自动检查服务运行状态
  - 测试API连接
  - 显示数据统计
  - 提供启动命令参考

## 🚀 使用流程

### 场景1：开始改造新模块

```bash
# 1. 运行检查清单，了解需要完成的任务
./改造检查清单.sh "新模块名称"

# 2. 打开详细文档，按步骤进行改造
cat APS改造最佳实践.md

# 3. 改造过程中遇到问题，查看快速参考
cat 快速参考卡片.md

# 4. 改造完成后，检查服务状态
./check-services.sh

# 5. 在浏览器中测试功能
open http://localhost:5173/
```

### 场景2：快速查阅代码

```bash
# 查看快速参考卡片
cat 快速参考卡片.md

# 搜索特定代码片段
grep -n "createXxx" 快速参考卡片.md
```

### 场景3：排查问题

```bash
# 1. 检查服务状态
./check-services.sh

# 2. 查看常见问题解决方案
cat APS改造最佳实践.md | grep -A 10 "常见问题"

# 3. 查看快速修复方法
cat 快速参考卡片.md | grep -A 5 "常见错误"
```

## 🎯 核心要点速记

### 必须遵守的原则

1. **数据格式统一**：前后端数据格式必须一致
2. **统一API服务**：必须从`api-unified.js`导入
3. **软删除优先**：删除操作使用软删除
4. **端口配置一致**：前后端端口必须匹配
5. **完善错误处理**：所有操作都要有异常捕获

### 关键文件清单

```
后端（3个文件）：
├── controllers/xxxController.js    # 添加CRUD方法
├── services/xxxService.js          # 实现业务逻辑
└── routes/xxxRoutes.js             # 配置路由

前端（4个文件）：
├── utils/api.js                    # 模拟数据API
├── utils/api-client.js             # 真实API客户端
├── utils/api-unified.js            # 统一API服务（必须！）
└── views/xxx/XxxManage.vue         # UI组件
```

### 代码模板位置

- **后端模板**：`APS改造最佳实践.md` → "快速开始模板"章节
- **前端模板**：`APS改造最佳实践.md` → "快速开始模板"章节
- **关键代码片段**：`快速参考卡片.md` → "关键代码片段"章节

## ⚠️ 重要提醒

### 改造前必读

1. **阅读完整文档**：不要只看快速参考，先阅读完整文档
2. **理解双模式原理**：确保理解模式切换的机制
3. **备份现有代码**：改造前备份相关文件
4. **逐个模块改造**：不要同时改造多个模块

### 改造中必做

1. **使用检查清单**：逐项检查，不要遗漏
2. **测试两种模式**：真实API和模拟数据都要测试
3. **验证数据格式**：确保前后端数据格式一致
4. **检查错误处理**：所有操作都要有错误处理

### 改造后必测

1. **运行检查清单**：确保所有检查项都已完成
2. **测试CRUD操作**：查询、新增、编辑、删除都要测试
3. **测试模式切换**：在两种模式间切换测试
4. **检查日志记录**：确保操作都有日志记录
