# 综合看板卡片高度修复报告

## 📊 修复时间
2026-04-04

## ❌ 问题描述

### 问题现象
综合看板中最上面的三个卡片(产能负荷概览、订单状态统计、关键指标)高度不一致,放在一排不好看。

### 原因分析
三个卡片的内容高度不同:
1. **产能负荷概览**: 包含4条产线的进度条,高度由内容决定
2. **订单状态统计**: 包含图表容器,固定高度250px
3. **关键指标**: 包含4个指标卡片,高度由内容决定

---

## ✅ 解决方案

### 修复方法

#### 1. 统一卡片高度
为 `.stat-card` 添加固定高度和Flex布局:

```css
.stat-card {
  margin-bottom: 20px;
  height: 340px; /* 固定高度,使三个卡片一致 */
  display: flex;
  flex-direction: column;
}
```

#### 2. 卡片内容垂直居中
使用 `:deep()` 选择器修改卡片内部布局:

```css
.stat-card :deep(.el-card__body) {
  flex: 1;
  display: flex;
  flex-direction: column;
  justify-content: center;
}
```

#### 3. 产能负荷列表垂直居中
```css
.capacity-list {
  padding: 10px 0;
  flex: 1;
  display: flex;
  flex-direction: column;
  justify-content: center;
}
```

#### 4. 关键指标网格垂直居中
```css
.metrics-grid {
  display: grid;
  grid-template-columns: repeat(2, 1fr);
  gap: 20px;
  padding: 10px 0;
  flex: 1;
  display: flex;
  flex-direction: column;
  justify-content: center;
}
```

## 📊 修复细节

### 卡片高度设置
- **固定高度**: 340px
- **选择理由**: 基于图表容器高度(250px) + 上下边距 + 标题高度

### 布局方式
- **外层容器**: Flex列布局
- **卡片主体**: Flex列布局,垂直居中
- **内容区域**: Flex列布局,垂直居中

### 兼容性
- ✅ 使用标准CSS Flex布局
- ✅ 使用 `:deep()` 修改Element Plus组件内部样式
- ✅ 响应式设计保持不变

---

## 📝 注意事项

1. **高度选择**: 340px是基于内容估算的,可以根据实际效果调整
2. **内容溢出**: 如果内容过多,可能需要调整高度或添加滚动
3. **响应式**: 在小屏幕上可能需要调整高度或布局

## 🔧 可选优化

### 方案1: 动态高度
根据内容动态计算高度:

```css
.stat-card {
  height: auto;
  min-height: 340px;
}
```

### 方案2: 使用Grid布局
使用CSS Grid实现更灵活的布局:

```css
.dashboard > .el-row {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}

.stat-card {
  height: 340px;
}
```

### 方案3: 响应式高度
根据屏幕大小调整高度:

```css
@media (max-width: 1200px) {
  .stat-card {
    height: 280px;
  }
}

@media (max-width: 768px) {
  .stat-card {
    height: auto;
    min-height: 200px;
  }
}
```
