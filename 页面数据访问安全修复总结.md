## ❌ 问题描述

### 核心问题
多个页面在访问数据时没有进行安全检查,当数据为 `undefined` 时导致运行时错误。

### 常见错误模式
```javascript
// ❌ 错误: 没有检查数据是否存在
const order = data.erp_sales_orders.find(o => o.order_id === slot.order_id)
// 如果 data.erp_sales_orders 是 undefined,会报错:
// "Cannot read properties of undefined (reading 'find')"

// ❌ 错误: 没有检查数组是否为空
const line = productionLines.value.find(l => l.line_id === lineId)
// 如果 productionLines.value 是 undefined,会报错:
// "Cannot read properties of undefined (reading 'find')"
```

---

## ✅ 修复方案

### 统一修复模式

#### 模式1: 数据访问添加默认值
```javascript
// ✅ 正确: 添加空数组默认值
const orders = data.erp_sales_orders || []
const order = orders.find(o => o.order_id === slot.order_id)
```

#### 模式2: 工具函数添加安全检查
```javascript
// ✅ 正确: 添加安全检查
const getLineName = (lineId) => {
  if (!productionLines.value || !Array.isArray(productionLines.value)) {
    return lineId || '未知产线'
  }
  const line = productionLines.value.find(l => l.line_id === lineId)
  return line ? line.line_name : lineId
}
```

#### 模式3: 数据加载添加调试日志
```javascript
// ✅ 正确: 添加调试日志
const loadData = () => {
  console.log('🔄 开始加载数据...')
  const data = initializeExtendedData()
  console.log('📦 数据结构:', Object.keys(data))
  console.log('  - erp_sales_orders:', data.erp_sales_orders?.length || 0, '条')

  productionLines.value = data.sys_work_centers || []
  console.log('✅ 产线数据已加载:', productionLines.value.length, '条')
}
```

export const safeFind = (array, predicate, defaultValue = null) => {

}

export const safeFilter = (array, predicate) => {

    return []

}
