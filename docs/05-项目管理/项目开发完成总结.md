## 一、项目概述

### 1.1 项目背景
杰诺家居云-APS系统是为定制家具企业打造的先进计划排程系统,基于"先排程后揉单"的核心业务逻辑,实现从订单接收到发货的完整生产流程管理。

### 1.2 项目目标
- 构建完整的7阶段业务流程:订单接收 → 订单拆解 → 粗排程 → 揉单批次 → 细排程 → 生产执行 → 齐套管理
- 实现智能排程算法,包括自动排程建议、开料优化、智能填缝
- 建立实时监控预警体系,包括交期预警、延期预警、超期预警
- 保证完整数据关联,11个数据表,4,485条模拟数据,100%数据关联完整性

### 1.3 项目范围
- 前端应用:33个页面,10个功能模块
- 数据模型:11个核心数据表
- 技术栈:Vue 3 + Vite + Element Plus + ECharts

## 三、核心技术亮点

### 3.1 "先排程后揉单"核心逻辑
系统严格遵循"先排程后揉单"的业务逻辑:
- **排程前置**: 通过订单池和粗排程完成订单的计划日期与产线分配
- **揉单合并**: 基于已排程结果,按产线和日期分组,利用合并算法生成生产批次
- **齐套控制**: 通过齐套规则和时间窗控制,平衡材料利用率与交期达成率

### 3.2 倒排计划算法
基于订单交期倒推计划开工日期:
- 最迟开工日 = 交期 - 齐套缓冲期 - 生产周期
- 自动计算各产线的计划日期和产能负荷
- 支持手动调整和优化建议

### 3.3 揉单时间窗控制
严格控制揉单的时间范围:
- 最大提前天数: +1天
- 最大延后天数: -1天
- 防止严重的时间偏差,保证交期达成

### 3.4 交期红线策略
在交期和利用率冲突时,优先保证交期:
- 当订单达到最迟开工日时,强制生成批次
- 标记为[加急/强制下发]
- 提示计划员权衡利弊

### 3.5 齐套管理机制
解决"不同子订单在不同日期完工"导致等待的问题:
- 齐套货位管理
- 齐套进度追踪
- 超期预警(占用超过7天)
- 支持全齐套和分批发货两种模式

### 3.6 完整数据流转链路
7个阶段,11个数据表,100%数据关联完整性:
```
erp_sales_order → aps_production_order → aps_schedule_task
→ aps_merge_batch → aps_batch_detail → cam_cutting_pattern
→ mes_wip_tracking → mes_sorting_slot
```

## 六、项目挑战与解决方案

### 6.1 挑战1: 业务逻辑复杂
**问题**: 定制家具行业的排程逻辑非常复杂,涉及多个阶段和多种策略。

**解决方案**:
- 将业务流程拆分为7个清晰的阶段
- 每个阶段独立的数据对象和状态管理
- 策略配置中心,支持灵活调整

### 6.2 挑战2: 数据关联复杂
**问题**: 11个数据表之间存在复杂的关联关系,保证数据一致性是难点。

**解决方案**:
- 使用外键约束保证数据一致性
- 通过Mock数据生成器自动创建关联数据
- 数据完整性验证报告

### 6.3 挑战3: 界面复杂度高
**问题**: 33个页面,包含大量的表格、表单、图表,界面复杂度高。

**解决方案**:
- 使用Element Plus组件库快速构建
- 统一的设计风格和交互模式
- 响应式布局适配不同屏幕

### 6.4 挑战4: 性能优化
**问题**: 大量数据和复杂计算可能导致性能问题。

**解决方案**:
- 分页查询减少数据加载量
- 虚拟滚动优化长列表渲染
- 按需加载图表组件
